Sharding, partitioning, read scale

  1. Default: single Postgres + PgBouncer + composite (organization_id, …) indexes.
  2. Partition append-only: audit_log, notification history, by month.
  3. Read replica for reporting; connection pgsql_report in Laravel when introduced.
  4. Shard (Citus / per-tenant DB) only after sustained CPU / row SLO pressure — see implementation plan.
