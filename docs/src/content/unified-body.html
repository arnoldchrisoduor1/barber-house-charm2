Critical paths SVG

Booking create: Next → POST bookings → Laravel auth/scope → BookingService → Postgres transaction + availability → Horizon (SMS) → Soketi broadcast → 201.

M-Pesa: STK POST → Daraja → callback → HMAC + IP + Redis idempotency → update transactions → broadcast POS.

Chat: GET/POST messages → broadcast private-org.{id}.chat.{channel} → Echo + TanStack invalidate/patch.

Haus Connect: Next → Laravel session token → Node → provider → webhooks → call_logs.

§4–5 Next.js routing & client state

Marketing (marketing), auth (auth), dashboard (dashboard), portal (portal), public book (public). Middleware: session refresh, request id, optional tenant host hint (verify with API). Default RSC for read-mostly; client for calendar/POS/WebSockets. TanStack keys; URL owns filters; no duplicated entitlements.

§6–7 Laravel lifecycle & API

Modules: Auth, Tenancy, Billing, Booking, Pos, Crm, Staff, Inventory, Notifications, Reporting, Integrations — services + events; no cross-module writes on foreign models. REST nested resources; cursor pagination for hot lists; RFC7807 errors.

M-Pesa idempotency SVG

§8–9 Postgres & Redis (summary)

Shared schema + organization_id; composite indexes; expand/contract migrations; primary + read replica + materialized views; partition audit/notifications; shard only after PgBouncer + indexes + replicas exhausted.

Redis namespaces {env}:{org_id}:{feature}:{id}; sessions, HTTP cache tags, rate limits, idempotency SETNX, Redlock, Horizon queues; jitter + mutex against stampedes.

§10 Realtime

Soketi private org channels; Laravel broadcasting auth; Node for calls; all SMS/email via queues.

§11–12 Load, bots, WAF

Stateless PHP behind LB; PgBouncer; separate worker fleet; S3 for uploads. Chokes: DB pool, cron thundering herd, webhook retries, CSV exports async, N+1, retail spikes (products). Edge: OWASP CRS, Turnstile on book/auth, bot score; app rate limiters; signed embed JWT for widgets; webhook IP allowlists.

§13 Security baseline

  • TLS 1.2+, HSTS, optional mTLS internal
  • Secrets manager; least-privilege DB users
  • SAST/DAST in CI; Renovate
  • Audit log append-only + SIEM stream
  • PII minimize/mask in logs

§15 Observability & §16 Definition of done

OpenTelemetry traces; Prometheus/Grafana; structured JSON logs + request id; synthetic health checks. SLO examples: API 99.9%; booking p95; webhook success 99.95%.

DoD per slice: OpenAPI + regenerated client; FormRequests + Policies + tests; Next loading/error states + a11y; idempotent payment jobs documented; runbook for env vars; load-test ticket for hot paths.
